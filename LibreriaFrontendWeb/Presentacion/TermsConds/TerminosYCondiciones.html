Términos y Condiciones

Sistema de Gestión de Librería "Farolito"
Al utilizar este sistema, usted acepta cumplir con los siguientes términos y condiciones de uso.

1. Definiciones

  • Sistema: Se refiere al Sistema de Gestión de Librería "Farolito", una aplicación web desarrollada para la gestión integral de librerías.
  • Usuario: Persona autorizada que accede y utiliza el sistema con credenciales válidas.
  • Administrador: Usuario con permisos completos para gestionar el sistema, usuarios y configuraciones.
  • Datos: Toda la información almacenada en el sistema incluyendo libros, clientes, reservas, pedidos y transacciones.

2. Uso del Sistema

2.1 Acceso Autorizado
  • El acceso al sistema está restringido a usuarios autorizados con credenciales válidas.
  • Cada usuario es responsable de mantener la confidencialidad de sus credenciales de acceso.
  • Las credenciales de usuario no deben ser compartidas con terceros no autorizados.
  • Los usuarios deben notificar inmediatamente cualquier uso no autorizado de sus credenciales.
2.2 Uso Apropiado
  • El sistema debe utilizarse únicamente para los fines comerciales de gestión de librería.
  • Está prohibido utilizar el sistema para actividades ilegales o no autorizadas.
  • Los usuarios no deben intentar acceder a áreas del sistema para las cuales no tienen permisos.
  • Se prohíbe cualquier intento de comprometer la seguridad del sistema.

3. Gestión de Datos

3.1 Responsabilidad de Datos
  • Los usuarios son responsables de la exactitud y validez de los datos que ingresan al sistema.
  • Se debe verificar la información antes de confirmar cualquier transacción o registro.
  • Los datos de clientes deben ser tratados con confidencialidad y respeto a la privacidad.
  • La información financiera debe manejarse de acuerdo con las normativas vigentes.
3.2 Respaldo y Seguridad
  • Se recomienda realizar respaldos regulares de la información crítica.
  • Los datos sensibles están protegidos mediante medidas de seguridad apropiadas.
  • El acceso a la información está controlado mediante roles y permisos.
  • Se mantienen registros de auditoría de las operaciones realizadas en el sistema.

4. Funcionalidades del Sistema

4.1 Gestión de Inventario
  • El sistema permite la carga y actualización de información de libros.
  • Se pueden importar datos desde archivos Excel siguiendo los formatos establecidos.
  • Los precios y stock deben mantenerse actualizados para reflejar la realidad comercial.
  • Se proporciona control de existencias y alertas de stock bajo.
4.2 Sistema de Reservas
  • Las reservas deben procesarse de acuerdo con las políticas comerciales establecidas.
  • Los pagos a través de MercadoPago están sujetos a los términos de dicha plataforma.
  • Se debe confirmar la disponibilidad antes de aceptar una reserva.
  • Las entregas deben registrarse apropiadamente en el sistema.

5. Pagos y Transacciones

  • Los pagos procesados a través de MercadoPago están sujetos a los términos y condiciones de dicha plataforma.
  • Las transacciones en efectivo deben registrarse adecuadamente en el sistema.
  • Se deben emitir comprobantes para todas las transacciones realizadas.
  • Los reembolsos y devoluciones están sujetos a las políticas comerciales de la librería.
  • Todas las transacciones quedan registradas para fines de auditoría y contabilidad.

6. Limitaciones de Responsabilidad

  • El sistema se proporciona "tal como está" sin garantías explícitas o implícitas.
  • Los desarrolladores no se hacen responsables por pérdidas de datos causadas por mal uso del sistema.
  • Es responsabilidad del usuario mantener respaldos adecuados de la información crítica.
  • El sistema puede experimentar interrupciones por mantenimiento o actualizaciones.
  • Se recomienda verificar la funcionalidad en un entorno de prueba antes de usar en producción.

7. Soporte Técnico

  • El soporte técnico se proporciona para resolver problemas relacionados con el funcionamiento del sistema.
  • Las consultas sobre el uso del sistema deben dirigirse a través de los canales oficiales.
  • Se proporcionará asistencia para la resolución de errores técnicos documentados.
  • La capacitación de usuarios puede estar disponible bajo acuerdo separado.

8. Actualizaciones y Mantenimiento

  • El sistema puede recibir actualizaciones periódicas para mejorar funcionalidad y seguridad.
  • Se notificarán las actualizaciones importantes con antelación cuando sea posible.
  • Algunas actualizaciones pueden requerir cambios en los procedimientos de uso.
  • El mantenimiento programado se realizará en horarios que minimicen la interrupción del servicio.

9. Modificaciones de los Términos

  • Estos términos y condiciones pueden ser modificados en cualquier momento.
  • Los cambios importantes serán notificados a los usuarios con antelación razonable.
  • El uso continuado del sistema después de las modificaciones constituye aceptación de los nuevos términos.
  • Se recomienda revisar periódicamente estos términos y condiciones.

10. Terminación del Acceso

  • El acceso al sistema puede ser terminado por incumplimiento de estos términos.
  • Los administradores pueden suspender o eliminar cuentas de usuario cuando sea necesario.
  • Al terminar el acceso, el usuario debe devolver cualquier material confidencial.
  • Ciertas obligaciones de confidencialidad pueden persistir después de la terminación del acceso.
